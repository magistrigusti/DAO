# Жесткий запрет для агента

По умолчанию агент работает в режиме ТОЛЬКО ЧАТ И АНАЛИЗ.

Агенту запрещено:
- создавать файлы;
- удалять файлы;
- менять файлы;
- вызывать apply_patch;
- переписывать тесты;
- менять архитектуру;
- удалять контракты, wrappers, tests, scripts;
- "улучшать" проект по собственной инициативе.

Агент может менять код только если пользователь явно написал:
"РАЗРЕШАЮ ПРАВИТЬ ФАЙЛЫ:"
и перечислил конкретные пути.

Если пользователь просит "напиши код в чат", агент обязан дать код в чат
и не трогать файловую систему.

Если агент считает, что нужно изменить архитектуру, он сначала обязан:
1. назвать точные файлы;
2. объяснить риск;
